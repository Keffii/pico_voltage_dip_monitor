Frame 1 - Overview

Pico runtime overview

In display-only mode, the Pico keeps its attention on fast sensing and on-device feedback. It watches three voltage sources, decides whether each source is behaving normally, and uses that result to drive the OLED without adding storage work that could slow the critical path.

Signals enter

Read three local voltage inputs

The Pico samples three separate sources and also listens to local controls such as the OLED buttons and status indicators.

Fast path

Sample every 10 ms

The main loop reads the latest voltage values on a fixed cadence so short dips can be seen quickly instead of being averaged away.

Per source

Track each source separately

Each source keeps its own recent history, stability state, expected operating range, and current dip status.

Noise control

Ignore noise and confirm real dips

The detector checks signal quality, waits for the drop to persist, and waits again for recovery so noise does not become a false event.

On-device results

Drive the on-device display

The same sensing pipeline feeds the OLED so the user can see trends, active dips, and channel behavior directly on the Pico.

Frame 2 - Display-only runtime flow

How display-only mode decides what to show

The detector does not react to every raw fluctuation. It checks signal quality first, learns what stable voltage looks like for each source, then turns confirmed changes into display updates while keeping the fast sensing loop protected.

Critical timing split
10 ms raw sampling 100 ms median summaries

Fast raw readings are used for low-latency detection. Slower summaries give the display a calmer view of what the signal is doing over time.

1. Read the newest voltage value

The Pico samples all three sources on a fixed schedule and adds each reading to a short recent-history window.

2. Check whether the signal is trustworthy

It asks whether the reading is inside the allowed range and whether the recent window is stable enough to represent a real signal.

3. Keep learning a stable expectation

Only good-quality readings help the Pico learn what normal voltage looks like for each source.

4. Start a dip only when the drop persists

A brief fall below the learned expectation is not enough by itself. The drop must survive a hold period before it becomes a real dip.

5. End the dip only after stable recovery

Recovery uses hysteresis and a second hold period so the detector does not chatter if the signal bounces near the threshold.

What each source remembers

Independent state per voltage source

  • A short window of recent raw readings
  • A slower rolling view used for median summaries
  • A learned idea of stable voltage
  • Whether a dip is active right now
  • How low the current dip has gone
  • How long to wait before allowing a new dip
Why false positives stay under control

The detector rejects bad evidence

If the input disappears, floats, or becomes too noisy, the Pico can treat that as missing signal instead of a real electrical dip. That protects the OLED from misleading spikes.

Why optional offload helps

Protect the OLED experience

When available, a secondary worker can absorb display-related work so the 10 ms sensing loop stays focused on reading the signal and deciding what is real.
